# Eis Product Management

This is a general skill to help product managers and product leaders execute your tasks and deliver high quality management work, facilitating decisions and analysis.

Before anything, you need to read and maintain in mind all rules (if available) here: `rules/rules-prod.md`
## When to Use This Skill

- Analyze market, competitors, other products
- Analyze own product to give more insights and identify gaps and opportunities 
- Create strategic and tactical presentations 
- When user need to analyze data, indicators and metrics of any kind
- when user need to create specifications like:
    - PRD: Product Requirement Document
    - FRD: Feature Requirement Document
    - Write Epics, Stories and tech tasks
- Plan and breakdown and write items like Epics, Stories, Tasks
- Prioritze roadmaps, backlogs and short, medium and long term planning
- Write status reports about projects, works in progress
- Compile, organize and write informations in well formatted and structured documents  that will be shared
- support and assist the PM or product leader in any other task that may need in your duty like manager
## When to invoke this skill

This skill should be invoke when the user asks to **do real product work**, such as:

- “Create / refine a PRD / spec / business case / 1-pager”
- “Turn this idea into a roadmap” / “Outcome roadmap for X”
- “Design a discovery plan / interview script / experiment plan”
- “Define success metrics / OKRs / metric tree”
- “Position this product against competitors”
- “Design an AI-powered feature / GenAI agent / AI PM lifecycle”
- “Run a difficult conversation / feedback / 1:1 / negotiation”
- “Plan a product strategy / vision / opportunity assessment”

## Instructions and Commands
Use instructions and comands below when needed:

- Create, modify or clarify a PRD, follow instructions in `/commands/spec-prd.md`
- Create, modify or clarify a FRD, follow instructions in `/commands/spec-frd.md`

For other needs not specified or that don't have instructions or templates, apply good practices commonly used by Product Managers in market.

If the user asks to see the commands or workflows of this skill, cite all commands available with a concise description.

Scenario A: "Create a PRD for a todo app" → Full question workflow
Scenario B: "Create a PRD based on this doc: [...]" → Validation workflow  
Scenario C: "Edit this PRD: [...]" → Focused improvement workflow

Be aware when the user chat in natural language, to trigger commands using correct instructions:

**For PRDs:**
- "create a PRD about..."
- "new PRD for..."
- "edit the PRD..."

**For Issues (Stories/Tasks/Bugs):**
- "create a story about..."
- "new task for..."
- "create a bug..."

**For Epics:**
- "create an epic about..."
- "new epic for..."

**For FRDs:**
- "create an FRD about..."
- "new FRD for..."

**For Clarification:**
- "clarify the specification..."
- "clarify the PRD..."
